当前结论（2026-03-09）：
- 生产构建下，首页、搜索、详情、年/月/周统计页在“先访问后断网”的路径下可继续打开；未缓存搜索词会降级为明确错误提示。
- iOS / iPadOS Web Push 的前置条件已确认：必须是已添加到主屏幕的 Web App、具备 Service Worker、Push API、Notifications API，并且权限请求必须由用户手势触发。
- 当前项目已具备主屏安装与 Service Worker 基础条件，但尚未实现推送订阅、服务端推送和通知设置 UI。
- 不支持项与替代方案已明确：不依赖 Background Sync / Periodic Background Sync / 后台常驻轮询来补发离线写操作；当前采用“离线只读 + 联网后手动重试”的替代方案，并在搜索、详情页给出离线提示与写操作禁用状态。

- 已完成 iPhone 16 Pro 等效视口 `402x874` 的专项验证，并在生产构建下覆盖 `/home`、`/search?q=沙丘`、`/detail?movie=dune-2&from=home`、`/records`、`/records/month`、`/records/week`；各页均无横向溢出，底部导航可见且未超出视口。
- 详情页“新增观影记录”弹窗在 `402x874` 下高度为 426px，底部贴齐视口但未超出，可完整显示，无需内部下拉条。

- 已完成核心流程回归测试，覆盖以下链路：`搜索（blade） -> 详情页`、`详情页加入待看 -> 首页待看清单同步`、`新增观影记录 -> 自动转为已观看并移出待看`、`编辑观影记录分数`、`删除观影记录 -> 返回未记录状态`、`保存短评`、`年度统计 -> 月详情 -> 周详情 -> 影片详情`。
- 本轮 9.4 未发现功能性阻塞问题；在 Windows shell 输出里，自动化脚本打印中文时会出现 `?` 占位，这属于控制台编码表现，不作为页面功能缺陷记录。
- 验收结论：截至 `2026-03-10`，任务 9 范围内未发现阻塞交付的问题，当前版本通过测试与验收；后续如需继续推进，剩余事项应进入任务 10 的提交与变更追踪，而不是继续停留在任务 9。
