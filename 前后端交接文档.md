1. 项目一句话描述：  
   个人使用的移动优先观影档案 PWA，用于搜索影视条目、加入待看、记录观影、写短评、查看年/月/周统计。

2. 前端技术栈：  
   `Ionic React + TypeScript + Vite + vite-plugin-pwa + Workbox + @rdlabo/ionic-theme-ios26`。

3. 已完成页面：  
   - `/home`：首页  
   - `/search`：搜索页  
   - `/detail`：影视详情页  
   - `/records`：年度统计页  
   - `/records/month`：月详情页  
   - `/records/week`：周详情页  
   - `/create`：兼容路由，当前重定向到 `/search`

4. 核心数据实体：  
   - `MediaItem`：影视条目主体，支持 `movie | season`  
   - `MediaRatings`：平台评分  
   - `WatchLog`：观影记录  
   - `ShortReview`：短评  
   - `MediaUserState`：用户状态（`none | wishlist | watched`）  
   - `RecordsOverview`：统计概览  
   - `RankedMediaItem` / `MediaCardItem`：列表卡片类对象

5. 当前最核心的 3 个功能：  
   - 搜索影视条目并进入详情页  
   - 详情页维护个人状态：加入待看、添加/修改/删除观影记录、保存短评  
   - 年/月/周三层统计浏览

6. 团队情况：  
   个人使用。

7. 偏好的技术路线：  
   - 个人项目，简单优先，避免过度设计  
   - 先把接口契约定清楚，再接真实接口  
   - 后端返回结构化字段，前端负责展示拼装  
   - REST 风格优先  
   - 前端业务逻辑尽量轻，聚合值能后端给就后端给  
   - 当前阶段允许 mock 数据 + 本地存储过渡，后续再切真实服务

8. 是否需要登录/注册：  
   不需要。

9. 是否需要角色/权限：  
   不需要。

10. 已知的数据关系：  
   - 一个 `MediaItem` 对应零到多条 `WatchLog`  
   - 一个 `MediaItem` 对应零到一条 `ShortReview`  
   - 一个 `MediaItem` 对应零到一份 `MediaRatings`  
   - 一个 `MediaItem` 对应一份 `MediaUserState`  
   - 统计数据主要由 `WatchLog + MediaItem` 聚合得到  
   - 搜索结果、榜单项、周内每日影片列表都基于 `MediaCardItem` / `RankedMediaItem` 这类轻量对象  
   - 电视剧按“单季”处理，每一季作为一个独立 `MediaItem`

11. 是否需要文件上传/下载：  
   当前不需要文件上传。  
   文件下载当前也不是核心需求；如要做，更像是后续“导出观影报告/海报长图”的增强项，暂不作为本阶段必需能力。

12. 是否需要搜索/分页/筛选：  
   - 搜索：需要，而且是核心功能  
   - 搜索容错：需要，接口中已预留 `suggestions`  
   - 分页：接口结构已预留分页对象，但前端当前主要是单页结果展示  
   - 筛选：当前前端未实现，不是第一优先级

13. 预计用户量：  
   个人项目量级，预计 `1 ~ 10` 人以内，默认按单用户或极低并发考虑。

14. 部署目标：  
   移动优先 PWA，重点面向 iPhone / iOS 主屏安装体验；同时保留桌面浏览器可访问能力。

## 补充说明

- 当前前端已通过 mock 数据把核心页面和状态流转跑通，但还没有切到真实后端。
- 当前本地还在使用 `localStorage` 承接待看、观影记录、短评等写操作，后续接后端时需要迁移到正式接口。
